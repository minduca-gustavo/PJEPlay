Quando o storage muda, cada chave dentro de `mudancas` não é o valor diretamente — é um objeto com dois campos:

```js
{
    oldValue: false,  // o que era antes
    newValue: true    // o que virou agora
}
```

Então `mudancas['rota_dadosProntos'].newValue` é o valor **depois** da mudança.

---

**`=== true`**

Compara o valor novo com `true`. Só entra no `if` quando a mudança foi para `true`. Quando o listener dispara pelo `false`, essa comparação falha e passa direto.
